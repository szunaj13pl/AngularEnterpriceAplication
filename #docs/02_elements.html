Exposing an Angular Component as an Custom Elements

In this example you will expose an Angular Component as an Custom Element and dynamically load it.

  1. Look into your package.json and find out that the package @angular/elements has been installed (when starting from scratch, this can be done using ng add @angular/elements which is also adding the needed polyfills).

  2. Have a look into the flight-app's flight-cancelling folder and inspect its contents. It contains a module, a simple component as well as two service files.

  3. Find out the the FlightCancellingService in the file flight-cancelling.service is forwarding to the DefaultFlightCancellingService. This means that everyone who is requesting a FlightCancellingService gets an DefaultFlightCancellingService.

  4. Open the flight-cancelling.module file and make sure that the FlightCancellingComponent is registered as an entry component.

    Show code 

    @NgModel({
    imports: [
        [...]
    ],
    declarations: [
        FlightCancellingComponent
    ],
    entryComponents: [
        FlightCancellingComponent
    ],
    [...]
})
export class FlightCancellingModule { 

    constructor(private injector: Injector) {

        let customElement = createCustomElement(FlightCancellingComponent, {
        injector: this.injector
        });

        customElements.define('flight-cancelling', customElement);

    }
}

  5. Use the modules constructor to wrap the FlightCancellingComponent as a custom element and register it under the name flight-cancelling with the browser.

  6. Open the file default-flight-cancelling.service.ts and implement its show method so that it dynamically creates an element with the name flight-cancelling.

    Show code 

    const flightCancelling = document.createElement('flight-cancelling');

flightCancelling['flightId'] = flightId;
flightCancelling.addEventListener('closed', () => document.body.removeChild(flightCancelling));

document.body.appendChild(flightCancelling);

  7. Open the file sidebar.component.ts. Inject the FlightCancellingService into the constructor. Also, create a cancel methods the delegates to the FlightCancellingService's show method. Pass a dummy parameter to the show method (e. g. show(7))

  8. Open the file sidebar.component.html and add a menu item that calls cancel.

    Show code 

    <li>
  <a (click)="cancel()">
    <i class="ti-arrow-top-right"></i>
    <p>Cancel Flights</p>
  </a>
</li>

  9. Start your solution and test it by clicking at Cancel Flights.

Dynamically loading a Micro App

This example demonstrates how to dynamically load a ÂµApp that is exposed as a Web Component.

  1. Have a look into the flight-app's assets folder. Find there a micro-apps folder with a bundle. This bundle contains a micro app that could by deployed separately.

  2. Open the sidebar.component.html and add an menu item Your Miles. It shell call a load method when clicked.

    Show code 

    <li>
<a (click)="load()">
    <i class="ti-arrow-top-right"></i>
    <p>Your Miles</p>
</a>
</li>

  3. Open the sidebar.component.ts file, inject the MicroAppLoaderService into the constructor and implement a load method which calls the MicroAppLoaderService's load method.

  4. In the shared folder, create a micro-app-loader.service.ts file with a load method. This method should create a script-tag for the micro app as well as a client-a element. Also append client-a to the hash so that it starts routing:

    @Injectable({ providedIn: 'root' })
export class MicroAppLoaderService {

    [...]
    
    state: 'not_loaded' | 'show' | 'hide' = 'not_loaded';

    load(): void {

        if (this.state === 'show') {
            location.hash = '';
            this.state = 'hide';
        }
        else if (this.state === 'hide') {
            location.hash = 'client-a/page1';
            this.state = 'show';
            return;
        }
        else {
            const script = document.createElement('script');
            script['src'] = 'assets/micro-apps/client-a.bundle.js';
            document.body.appendChild(script);

            const clientA = document.createElement('client-a');
            const content = document.getElementsByClassName("content")[0];
            content.appendChild(clientA);

            location.hash = 'client-a/page1';
            this.state = 'show';
        }
    }
}

  5. Start your solution and test it.
